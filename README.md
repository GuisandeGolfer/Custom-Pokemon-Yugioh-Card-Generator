# Card Creator

AI-powered custom trading card generator that creates Pokemon and Yu-Gi-Oh style cards for your life events, trips, and memorable moments.

## Overview

Card Creator lets you turn any life event into a collectible trading card. Enter details about a trip, party, or milestone, choose a visual theme and element type, and the app generates a fully styled card complete with AI-generated artwork, a creative card name, and flavor text — all rendered in an authentic Pokemon or Yu-Gi-Oh card template that you can download as a PNG.

## Features

- **Two card styles** — Toggle between Pokemon and Yu-Gi-Oh card layouts, each with authentic styling
- **AI-generated card art** — Powered by Google's Nano Banana Pro (`gemini-3-pro-image-preview`) image generation model
- **AI-generated text** — Card names and flavor text generated by Gemini 2.0 Flash, weaving your event details into card-game narrative
- **6 element types** — Fire, Water, Earth, Wind, Light, and Dark — each with distinct color schemes
- **7 aesthetic themes** — Epic, Cute, Retro, Dark, Tropical, Fantasy, Cyberpunk, or enter your own custom theme
- **Event details embedded** — Date, location, and people involved are incorporated into the card design
- **PNG export** — Download your finished card as a high-resolution PNG image
- **Responsive layout** — Works on desktop and mobile

## Getting Started

### Prerequisites

- Node.js 18+
- A [Google AI Studio](https://aistudio.google.com/) API key with access to Gemini models

### Installation

```bash
# Clone the repository
git clone <repo-url>
cd custom_card_app

# Install dependencies
npm install
```

### Configuration

Create a `.env` file in the project root:

```
GEMINI_API_KEY=your_api_key_here
```

### Running the App

```bash
npm run dev
```

This starts both the Vite dev server (port 5173) and the Express backend (port 3001) using concurrently. Open `http://localhost:5173` in your browser.

### Production Build

```bash
npm run build
npm run preview
```

## How It Works

1. **User fills the form** — Card style (Pokemon/Yu-Gi-Oh), event name, date, location, people, theme, and element type
2. **Frontend calls `/api/generate-card`** — Sends all form data to the Express backend
3. **Backend makes two parallel Gemini API calls:**
   - Text generation (Gemini 2.0 Flash) — returns card name, description, level, and stats as JSON
   - Image generation (Nano Banana Pro) — returns base64-encoded card art
4. **Card renders in the browser** — The chosen template (Pokemon or Yu-Gi-Oh) displays the AI-generated content with element-appropriate styling
5. **User downloads the card** — html-to-image converts the card DOM node to a high-res PNG

## API Endpoints

### `POST /api/generate-card`

Generates a complete card with AI art and text.

**Request body:**
```json
{
  "cardStyle": "pokemon",
  "eventName": "Trip to Tokyo",
  "eventDate": "2026-04-15",
  "location": "Tokyo, Japan",
  "people": "Diego, Maria, Carlos",
  "theme": "cyberpunk",
  "elementType": "fire"
}
```

**Response:**
```json
{
  "cardName": "Neon Sakura Guardian",
  "description": "A warrior born from the electric streets of Tokyo...",
  "level": 7,
  "stats": { "hp": 120, "attack": 80, "defense": 60 },
  "imageBase64": "data:image/png;base64,...",
  "elementType": "fire"
}
```

### `GET /api/health`

Returns `{ "status": "ok" }` — useful for verifying the backend is running.

## Notes

- Image generation typically takes 10-20 seconds — a loading overlay is shown during this time
- The `html-to-image` package is pinned to v1.11.11 for stability
- The `.env` file is gitignored to keep your API key safe
